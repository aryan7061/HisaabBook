fix(account-settings): unnecessary refetch, unsaved changes, validation

- Tied useForm's query to the drawer's open state (queryOptions.enabled),
  fixing a bug where user data was fetched on every single page load
  regardless of whether the drawer was ever opened
- Fixed unsaved edits persisting after closing and reopening the drawer
  (Ant Design Drawer doesn't unmount on close, so form state survived);
  now resets form fields on discard
- Added a confirm dialog before discarding touched-but-unsaved changes
- Fixed incorrect placeholder ("Timezone" on the Phone field) and added
  the missing Timezone field, which the schema already supports
- Added validation: required Name, email format check, phone number
  pattern
- Added Cancel button alongside Save; disabled mask-click-to-close to
  prevent accidental data loss
